import java.io.DataOutputStream;
import java.net.Socket;

public class MyClient {
   public static void main(String[] args) {
        try {
            Socket s = new Socket("localhost", 6666);
            DataOutputStream dout = new DataOutputStream(s.getOutputStream());
            dout.writeUTF("Hello Server");
            dout.flush();
            dout.close();
            s.close();

        } catch (Exception e) {
            System.out.println(e);
        }
    }
}

/*
Simple Explanation (Client):
1) Socket("localhost", 6666): creates a TCP connection from client to server on the same machine.
    - localhost means "this computer".
    - 6666 is the port number where the server is listening.
2) DataOutputStream: wraps the socket output so we can send Java-friendly data types.
    - It is easier than sending raw bytes manually.
3) writeUTF("Hello Server"): sends one string message in modified UTF format.
    - The method includes length information, so receiver knows how many bytes to read.
4) flush(): pushes any buffered data immediately to the network.
5) close() calls: closes stream and socket to avoid memory/resource leaks.
    - If not closed, program may keep unused connections open.

How it works:
- Start server first.
- Run client.
- Client connects, sends one message, and exits.

Data Flow:
- Client side: String -> writeUTF -> bytes on socket.
- Server side: bytes on socket -> readUTF -> String.

Important note:
- This client sends only one message and then stops.
- For chat-like communication, you would use a loop.
*/


import java.io.DataInputStream;
import java.net.ServerSocket;
import java.net.Socket;

public class MyServer {
    public static void main(String[] args) {
        try {
            ServerSocket ss = new ServerSocket(6666);
            Socket s = ss.accept(); // establishes connection
            DataInputStream dis = new DataInputStream(s.getInputStream());
            String str = dis.readUTF();
            System.out.println("message = " + str);
            ss.close();

        } catch (Exception e) {
            System.out.println(e);
        }
    }
}

/*
Simple Explanation (Server):
1) ServerSocket(6666): starts a TCP server and binds it to port 6666.
    - This makes the server ready to accept incoming client connections.
2) accept(): blocks (waits) until some client connects.
    - After connection, it returns a Socket dedicated to that client.
3) DataInputStream: wraps socket input to easily read Java data types.
4) readUTF(): receives the string sent by client using writeUTF.
    - This is why both client and server should use matching methods.
5) println: prints received message to console.
6) close(): closes the server socket and releases network resources.

How it works:
- Server starts and waits.
- Client connects and sends "Hello Server".
- Server reads and prints that message, then program ends.

Data Flow:
- Server is passive first (waiting at accept).
- After connection, it reads incoming bytes and converts to string.

Important note:
- This server handles only one client and one message.
- To support multiple clients, usually a loop and threads are used.
*/

Calculator.java
import java.rmi.Remote;
import java.rmi.RemoteException;

public interface Calculator extends Remote {
    int add(int a, int b) throws RemoteException;
    int subtract(int a, int b) throws RemoteException;
    int multiply(int a, int b) throws RemoteException;
    int divide(int a, int b) throws RemoteException;
}

/*
 * EXPLANATION:
 * This file defines the Remote Interface (`Calculator`).
 * 
 * 1. It extends `java.rmi.Remote`, which acts as a marker to show that the methods inside can be called remotely.
 * 2. Every method must throw a `java.rmi.RemoteException` to handle potential network or communication failures.
 * 3. This acts as a contract/blueprint: the Server will implement these methods, and the Client will call them.
 */

CalculatorServer.java
import java.rmi.RemoteException;
import java.rmi.registry.LocateRegistry;
import java.rmi.registry.Registry;
import java.rmi.server.UnicastRemoteObject;

public class CalculatorServer extends UnicastRemoteObject implements Calculator {
    private static final long serialVersionUID = 1L;

    public CalculatorServer() throws RemoteException {
        super();
    }

    public int add(int a, int b) throws RemoteException {
        return a + b;
    }

    public int subtract(int a, int b) throws RemoteException {
        return a - b;
    }

    public int multiply(int a, int b) throws RemoteException {
        return a * b;
    }

    public int divide(int a, int b) throws RemoteException {
        if (b == 0)
            throw new RemoteException("Cannot divide by zero");
        return a / b;
    }

    public static void main(String[] args) {
        try {
            CalculatorServer obj = new CalculatorServer();
            Registry registry = LocateRegistry.createRegistry(1100);
            registry.rebind("Calculator", obj);
            System.out.println("Server is running...");
        } catch (Exception e) {
            System.out.println(e);
        }
    }
}

/*
 * EXPLANATION:
 * This file is the Server Implementation (`CalculatorServer`).
 * 
 * 1. Extending `UnicastRemoteObject` ensures this class can export itself to accept incoming remote calls from clients automatically.
 * 2. It implements the `Calculator` interface, providing the actual logic for add, subtract, multiply, and divide.
 * 3. A constructor declaring `RemoteException` is required because `UnicastRemoteObject` constructor throws it.
 * 4. `LocateRegistry.createRegistry(1100)`: Automatically starts an RMI registry on port 1100, saving us from using the external `rmiregistry` command manually.
 * 5. `registry.rebind("Calculator", obj)`: Registers our actual implementation object in the registry with the name "Calculator", making it discoverable.
 */

import java.rmi.registry.LocateRegistry;

public class client1 {

            System.out.println("10 + 5 = " + calc.add(10, 5));
            System.out.println("10 - 5 = " + calc.subtract(10, 5));
            System.out.println("10 * 5 = " + calc.multiply(10, 5));
            System.out.println("10 / 5 = " + calc.divide(10, 5));

        } catch (Exception e) {
            System.out.println(e);
        }
    }
}

/*
 * EXPLANATION:

 * 3. `registry.lookup("Calculator")`: Searches the registry for the name we
 * registered the server under ("Calculator").
 * 4. It casts the stub/reference returned by the lookup into the `Calculator`
 * interface.
 * 5. Then, it invokes `.add`, `.subtract`, etc., exactly as if it were a local
 * object, abstracting the network communication details behind the interface.
 */

from xmlrpc.server import SimpleXMLRPCServer

def add(a, b):
    return a + b

def sub(a, b):
    return a - b

def mul(a, b):
    return a * b

server = SimpleXMLRPCServer(("localhost", 8000), allow_none=True)
print("RPC Server running on port 8000...")
server.register_function(add, "add")
server.register_function(sub, "sub")
server.register_function(mul, "mul")

server.serve_forever()

    - "sub" can be called by client as proxy.sub(...)
    - "mul" can be called by client as proxy.mul(...)
4) serve_forever() keeps server running and listening for requests continuously.

How request/response flow works:
- Client sends method name + arguments in XML format over HTTP.
- Server matches method name to registered function.
- Function runs and return value is sent back to client.

Important note:

/*

1) MulticastSocket socket = new MulticastSocket();

2) InetAddress.getByName("230.0.0.1"):
    - Defines the multicast group IP address.
3) int port = 5000:
    - Defines the group port where receivers are listening.
4) message.getBytes():
    - Converts text message into bytes for network transfer.
5) DatagramPacket(...):
    - Wraps data + destination address + destination port into one packet.
6) socket.send(packet):
    - Sends the message to all receivers that joined this multicast group.
7) socket.close():
    - Releases socket resources after sending.

How it works:
- Sender does not connect to a single client.
- It sends one UDP packet to the multicast group address.
- Any active receiver in that group can get the message.

Important note:
- This sender sends only one message and exits.
- For continuous sending, put message creation and send in a loop.
*/


import java.io.IOException;
import java.net.DatagramPacket;
import java.net.InetAddress;
import java.net.InetSocketAddress;
import java.net.MulticastSocket;
import java.net.NetworkInterface;

public class GroupReceiver {
    public static void main(String[] args) throws IOException {
        int port = 5000;
        InetAddress groupIp = InetAddress.getByName("230.0.0.1");
        InetSocketAddress group = new InetSocketAddress(groupIp, port);

        // Pick the interface you want to receive multicast on
        NetworkInterface nif = NetworkInterface.getByInetAddress(InetAddress.getLocalHost());

        try (MulticastSocket socket = new MulticastSocket(port)) {
            socket.joinGroup(group, nif);

            while (true) {
                byte[] data = new byte[1024];
                DatagramPacket packet = new DatagramPacket(data, data.length);
                socket.receive(packet);
                String message = new String(packet.getData(), 0, packet.getLength());
                System.out.println("Received message: " + message);
            }
        }
    }
}

/*
Simple Explanation (GroupReceiver):
1) groupIp and port define which multicast group to listen to.
2) InetSocketAddress group = new InetSocketAddress(groupIp, port):
    - Creates a socket address for the multicast group.
3) NetworkInterface nif = NetworkInterface.getByInetAddress(InetAddress.getLocalHost()):
    - Selects a network interface used for joining the multicast group.
4) MulticastSocket socket = new MulticastSocket(port):
    - Opens receiver socket and binds it to port 5000.
5) socket.joinGroup(group, nif):
    - Joins multicast group using modern API (non-deprecated).
6) socket.receive(packet):
    - Waits for incoming UDP multicast packet.
7) new String(packet.getData(), 0, packet.getLength()):
    - Converts received bytes back into readable message text.

How it works:
- Start receiver first so it can join the group.
- Run sender to transmit message.
- Receiver gets packet and prints it.

Important note:
- while (true) keeps receiver running continuously.
- Stop manually (for example, Ctrl+C) when done.

class BullyElection:
  def __init__(self, processes):
    self.processes = processes  # list of process IDs
    self.alive = {p: True for p in processes}
    self.coordinator = max(processes)

  def crash(self, pid):
    self.alive[pid] = False
    print(f"Process {pid} crashed.")

  def start_election(self, pid):
    print(f"\nProcess {pid} starts election")
    higher = [p for p in self.processes if p > pid and self.alive[p]]

    if not higher:
      self.coordinator = pid
      print(f"Process {pid} becomes new Coordinator")
    else:
      print(f"Process {pid} sends Election to {higher}")
      new_leader = max(higher)
      self.start_election(new_leader)

  def show_coordinator(self):
    print("Current Coordinator:", self.coordinator)


processes = [1, 2, 3, 4, 5]
bully = BullyElection(processes)
bully.show_coordinator()
bully.crash(5)  # coordinator crashes
bully.start_election(2)
bully.show_coordinator()


"""
Simple Explanation (Bully Election):
1) Every process has a unique ID. Highest alive ID should be coordinator.
2) __init__ stores all process IDs, marks all as alive, and picks highest as initial coordinator.
3) crash(pid) marks a process as failed.
4) start_election(pid) is called when a process starts election.
   - It checks alive processes with higher IDs.
   - If none exist, current process becomes coordinator.
   - If higher processes exist, highest one continues election.
5) show_coordinator() prints the current coordinator.

How this demo runs:
- Initial coordinator is process 5.
- Process 5 crashes.
- Process 2 starts election.
- Election moves upward and process 4 becomes new coordinator.

# Ring Election Algorithm Simulation

class RingElection:
	def __init__(self, processes):
		self.processes = processes
		self.alive = {p: True for p in processes}
		self.coordinator = max(processes)

	def crash(self, pid):
		self.alive[pid] = False
		print(f"Process {pid} crashed.")

	def start_election(self, starter):
		print(f"\nProcess {starter} starts election")

		active_list = []
		n = len(self.processes)
		index = self.processes.index(starter)

		for i in range(n):
			pid = self.processes[(index + i) % n]
			if self.alive[pid]:
				active_list.append(pid)

		print("Active Processes:", active_list)

		new_leader = max(active_list)
		self.coordinator = new_leader
		print(f"Process {new_leader} elected as Coordinator")

	def show_coordinator(self):
		print("Current Coordinator:", self.coordinator)




# Example

ring = RingElection(processes)
ring.show_coordinator()
ring.crash(4)
ring.start_election(2)

Simple Explanation (Ring Election):
1) Every process has a unique ID and they are logically arranged in a ring.

4) start_election(starter) begins election from one process.
5) The algorithm traverses processes in ring order starting from starter.
6) Only alive processes are collected in active_list.
7) Highest ID from active_list is elected as the new coordinator.

Exp 6 Mutual Exclusion

import time

class Process:
    def __init__(self, pid, needs_cs=False):
        self.pid = pid
        self.needs_cs = needs_cs

    def execute(self):
        if self.needs_cs:
            print(f"Process P{self.pid} has the token and enters Critical Section.")
            time.sleep(1)
            print(f"Process P{self.pid} exits Critical Section.\n")
            self.needs_cs = False
        else:
            print(f"Process P{self.pid} has the token but does not need Critical Section.\n")
            time.sleep(0.5) 

def token_ring(processes, cycles):
    token = 0 # token starts with first process
    for c in range(cycles):
        print(f"--- Cycle {c+1} ---")
        current_process = processes[token]
        current_process.execute()
        # pass token to next process in ring
        token = (token + 1) % len(processes)

# Main program
if __name__ == "__main__":
    n = int(input("Enter number of processes: "))
    processes = []
    print("Enter 1 if process wants to enter critical section, otherwise 0")
    for i in range(n):
        need = int(input(f"Does Process P{i} need CS? (1/0): "))
        processes.append(Process(i, need == 1))
    cycles = int(input("Enter number of token cycles: "))
    token_ring(processes, cycles)


"""
EXPLANATION:
This code implements the Token Ring Algorithm used for mutual exclusion in distributed systems.

1. Process Class: Represents a node in the ring. Each process has an ID (`pid`) and a flag (`needs_cs`) to indicate if it wishes to enter the Critical Section (CS).
2. execute() Method: Simulates the process's turn when it receives the token.
   - If it requested the CS, it "enters" it (simulated by a 1-second pause), finishes its task, exits, and resets its request flag.
   - If it didn't request the CS, it simply acknowledges holding the token and quickly passes it on.
3. token_ring() Function: Manages the cyclical passing of the token. It starts at process 0 and iterates through the token cycles. The expression `(token + 1) % len(processes)` ensures that the token loops back to the first process after reaching the last one.
4. Main Execution: Prompts the user to set up the environment, including the number of processes, which processes require the CS, and the total number of ring cycles to simulate.

def calculate_task_assignment():
    n = int(input("Enter number of tasks: "))
    m = int(input("Enter number of processors: "))
    
    print("\nEnter execution cost matrix:")
    exec_cost = []
    for i in range(n):
        row = list(map(int, input(f"Enter costs for Task {i} on {m} processors: ").split()))
        exec_cost.append(row)
        
    print("\nEnter communication cost matrix:")
    comm_cost = []
    for i in range(n):
        row = list(map(int, input(f"Enter communication cost for Task {i} with other tasks: ").split()))
        comm_cost.append(row)
        
    print(f"\nEnter assigned processor for each task (0 to {m-1}):")
    assignment = list(map(int, input().split()))
    
    execution_time = 0
    communication_time = 0
    
    # Calculate execution cost
    for i in range(n):
        execution_time += exec_cost[i][assignment[i]]
        
    # Calculate communication cost
    for i in range(n):
        for j in range(i + 1, n):
            if assignment[i] != assignment[j]:
                communication_time += comm_cost[i][j]
                
    total_assignment_time = execution_time + communication_time
    
    print("\nExecution Time =", execution_time)
    print("Communication Time =", communication_time)
    print("Total Assignment Time =", total_assignment_time)

# Main
if __name__ == "__main__":
    calculate_task_assignment()

"""
EXPLANATION:
This code calculates the total cost (time) of a specific assignment of tasks to processors in a distributed system.

1. Input Matrices formulation:
   - `exec_cost`: Represents the time it takes to execute each task on each processor. Size is (n x m) where n is tasks and m is processors.
   - `comm_cost`: Represents the communication time penalty if two tasks communicate and run on DIFFERENT processors. Size is typically (n x n).

2. Task Assignment:
   - The `assignment` array maps each task (by its index) to a specific processor (value).

3. Cost Calculation:
   - Execution Cost: Adds up the execution time for each task on its assigned processor.
   - Communication Cost: Adds communication penalties between pairs of tasks only if they are assigned to different processors (`assignment[i] != assignment[j]`).

4. Output: The total assignment time is given by the sum of total execution time and total communication penalties.
"""
